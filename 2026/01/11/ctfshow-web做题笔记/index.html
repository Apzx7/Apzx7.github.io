### 命令执行

以下函数用于执行外部系统命令（如Shell命令）；

| **函数**                                                     | **返回值/行为**                                              | **示例**                                            | **特点**                                   |
| ------------------------------------------------------------ | ------------------------------------------------------------ | --------------------------------------------------- | ------------------------------------------ |
| **`system()`**                                               | 返回命令输出的最后一行，并直接打印结果                       | `system('ls -l', $retval);`                         | 自动刷新输出缓冲区，适合需要实时显示的场景 |
| **`exec()`**                                                 | 返回最后一行输出，需通过数组参数捕获完整结果                 | `exec('ls', $output, $status);`                     | 无直接输出，需手动处理结果                 |
| **`passthru()`**                                             | 无返回值，直接输出原始二进制结果（如图像）                   | `passthru('cat image.jpg');`                        | 适用于二进制数据流传输                     |
| **`shell_exec()`**                                           | 返回完整输出字符串（含错误信息）                             | `$result = shell_exec('whoami');`                   | 需通过`echo`显示结果                       |
| **`popen()`**                                                | 返回文件指针，需用`fread()`读取结果                          | `$handle = popen('ls', 'r'); fread($handle, 1024);` | 适合处理大体积输出或流式命令               |
| **反引号 ``**         | 与`shell_exec()`等效                       | ``$output = `ls`;``配合打印函数如echo，printf               | 语法简洁，但禁用`shell_exec`时失效 |                                                              |                                                     |                                            |
| proc_open                                                    | 执⾏⼀个命令，并且打开⽤来输⼊/输出的⽂件指针。              |                                                     |                                            |
| pcntl_exec()                                                 | 在当前进程空间执⾏指定程序，当发⽣错误时返回 false ，没有错误时没有回。 |                                                     |                                            |



#### 常见代码执行
| **函数**                | **行为**                                                  | **示例**                                                     | **风险场景**                            |
| ----------------------- | --------------------------------------------------------- | ------------------------------------------------------------ | --------------------------------------- |
| **`eval()`**            | 执行字符串中的PHP代码,<br>接收字符串，有时可与inplode配合 | `eval('echo "Hello, ".$_GET["name"];');`                     | 需分号结尾，变量会保留到主脚本          |
| **`assert()`**          | 调试断言，但可执行代码                                    | `assert('system("whoami")');`                                | 传统用法（PHP 7.2后不推荐）             |
| **`preg_replace()`**    | 用`/e`修饰符时，替换字符串可执行代码                      | `preg_replace('/test/e', 'system("ls")', 'test');`           | PHP 5.5+已废弃`/e`                      |
| **`create_function()`** | 创建匿名函数，但参数未过滤可执行代码                      | `$func = create_function('', 'echo $_GET["cmd"];'); $func();` | 易被注入（如`cmd=phpinfo();`）          |
| **回调函数**            | 通过函数名动态调用代码                                    | `call_user_func('system', $_POST['cmd']);`                   | 需结合危险函数名（如`system`）          |
| **动态函数调用**        | 变量函数名直接调用                                        | `$func = $_GET['f']; $func('id');`                           | 用户可控`$func`时风险高（如`f=system`） |

    • 无敏感关键词：未触发 flag|system|php|cat 等黑名单（正则过滤了这些关键词）
    • 伪协议绕过执行：php://filter 对文件内容进行 Base64 编码后再返回，避免直接执行 flag.php 中的代码（若直接包含 flag.php 会执行而非显示源码）

web39

加了个.php后缀

这儿，已经闭合了，不受这个后缀影响；

  • 在正常情况下，include函数会尝试包含一个以.php为后缀的文件。但在这种情况下，由于使用了data://协议，$c的值直接是一个嵌入的数据流，而不是一个文件名。
  • 因此，include函数会直接解析这个数据流作为PHP代码执行，而不会去查找一个文件，所以不需要.php后缀。

小结

s2-001漏洞：
该漏洞因为用户提交表单数据并且验证失败时，后端会将用户之前提交的参数值使用 OGNL 表达式 %{value} 进行解析，然后重新填充到对应的表单数据中。例如注册或登录页面，提交失败后端一般会默认返回之前提交的数据，由于后端使用 %{value} 对提交的数据执行了一次 OGNL 表达式解析，所以可以直接构造 Payload 进行命令执行。WebWork 2.1+ 和 Struts 2 的“altSyntax”功能允许将 OGNL 表达式插入到文本字符串中并进行递归处理。
OGNL是Object Graphic Navigation Language(对象图导航语言)的缩写，它是一个开源项目。Struts2框架使用OGNL作为默认的表达式语言。
作用：取值，获取javaBean中的属性，获取List或者数组元素，获得Map的键值对，还可以执行逻辑运算。
要求：我们必须把OGNL表达式写在struts的标签中。

对该漏洞进行分析，我们需要知道如下内容：

  1. struts2是怎么运作的
  2. Java的反射机制和Java的类加载机制和Java的动态代理
  3. Ognl表达式
  4. IDEA调试方法
